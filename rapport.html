5. Description technique

Base de données MySQL

La base de données est organisée en plusieurs tables principales :

TableDescription
utilisateurStocke les informations des utilisateurs (id, nom, email, mot de passe hashé, etc.)
TrajetContient les informations relatives au trajet de l'utilisateur(point de depart, d'arrivé, heure de depart, nombres de places:si l'utilisateur est un conducteur)
Demande contient les informations relatives au trajet de l'utilisateur et les coordonnées geographique de son point de depart et d'arrivé
MatchingContient des informations relatives à l'ofres et à la demande
messagesHistorique des échanges entre utilisateurs
Reservationhistorique des futurs trajets conducteur-passagers apres le matching

Backend Flask

Le serveur Flask gère les routes principales :

  • / - connexion
  • / - inscription
  • / - profil utilisateur
  • / - consultation et création d'offres
  • / - consultation et création de demandes
  • / - messagerie interne

Sécurité

  • Hashage des mots de passe avec bcrypt
  • Validation des formulaires côté client et serveur
  • Protection contre les injections SQL via ORM et requêtes préparées

5- Instruction de déploiement

Prérequis


Python 3.10+ (recommandé : 3.11) MySQL (ou MariaDB) Node.js (optionnel, pour un serveur frontend local) Git (pour cloner le projet)

. Cloner le projet

br git clone https://github.com/X045-lyse/PIL1_2425_24.git cd PIL1_2425_24

.. Installer et configurer la base de données MySQL


Créer la base de données et l’utilisateur :
CREATE DATABASE covoiturage CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'ifriuser'@'localhost' IDENTIFIED BY 'motdepassefort';
GRANT ALL PRIVILEGES ON covoiturage.* TO 'ifriuser'@'localhost';
FLUSH PRIVILEGES;

Vérifier/modifier la chaîne de connexion dans backend/app.py si besoin :
app.config['SQLALCHEMY_DATABASE_URI'] = 'mysql+pymysql://ifriuser:motdepassefort@localhost/covoiturage';

... Installer les dépendances Python (backend)


cd backend
python -m venv ../env
../env/Scripts/activate # (Windows)
# ou source ../env/bin/activate # (Linux/Mac)
pip install -r requirements.txt

.... Initialiser la base de données

python app.py
Les tables seront créées automatiquement au premier lancement grâce à db.create_all().br

..... Lancer le backend Flask

python app.py
Le backend sera accessible sur http://localhost:5000

...... Lancer le frontend


Option 1 : Ouvrir les fichiers HTML directement (pas recommandé pour les appels API)
Option 2 : Utiliser un serveur local (recommandé)
Avec VS Code (Live Server) : Clique droit sur frontend/index.html → "Open with Live Server"
Ou avec Python :
cd ../frontend
python -m http.server 5500
Le frontend sera accessible sur http://localhost:5500

....... Configuration CORS


Le backend autorise déjà les origines localhost et 127.0.0.1 sur les ports 5000 et 5500.
Si tu changes de port ou d’URL, adapte la liste dans backend/app.py :
CORS(
app, supports_credentials=True,
origins=[
"http://localhost:5500",
"http://127.0.0.1:5500",
"http://localhost:5000",
"http://127.0.0.1:5000"
]
)

........ Dépannage


Erreur CORS : vérifie la config CORS et que tu utilises bien le bon port/URL.
Erreur 404 sur une image : place l’image manquante dans frontend/images/.
Erreur de connexion MySQL : vérifie l’utilisateur, le mot de passe et que MySQL tourne.
